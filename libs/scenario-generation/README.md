# `scenario-generation`

A Python package for ingesting transportation network topologies from standard TNTP formats and generating stochastic demand and capacity scenarios for Graph Neural Network (GNN) surrogate modeling.

This package powers the data pipeline described in **"GUIDED Network-Agnostic Feature Initialization for Spatial Transferability in GNN-based Models"** (Scalese et al., 2026).

---

## Key Features

1. **Network Initialization (`scenarios initialize`)**:
   - Ingests raw TNTP network files (`net.tntp`, `trips.tntp`, `flows.tntp`, `nodes.tntp` or `<network>_nodes.geojson`).
   - Standardizes spatial projections and outputs a consolidated `<network>_master.gpkg` containing four layers: `nodes`, `links`, `od`, and `flows`.

2. **Scenario Generation (`scenarios generate`)**:
   - **Dataset A (Random Uniform Scaling)**: Simulates nominal daily traffic variations by applying random capacity reduction $\delta_r \sim U(0.5, 1.0)$ and uniform demand scaling $\delta_{ij} \sim U(0.5, 1.5)$.
   - **Dataset B (Dirichlet Distribution Shift)**: Simulates out-of-distribution, structurally novel demand patterns by applying random capacity reduction $\delta_r \sim U(0.5, 1.0)$ and redistributing total network demand via a symmetric Dirichlet distribution $\mathbf{p} \sim \text{Dir}(\alpha \cdot \mathbf{1}_K)$ with concentration parameter $\alpha = 0.05$.
   - **Deterministic & Parallelized**: Uses scenario-specific random seeds for 100% reproducible scenario generation and supports multi-core parallel processing.

## Directory Structure

### 1. Required Source Input Structure (`networks/`)
To initialize a network, raw files must be placed in a subdirectory under `networks/` matching the network name:

```text
code/
├── networks/
│   ├── anaheim/
│   │   ├── net.tntp                   # Link topology, capacity, and BPR parameters
│   │   ├── trips.tntp                 # Origin-Destination demand blocks
│   │   ├── flows.tntp                 # Ground-truth flow solutions
│   │   ├── nodes.tntp                 # Node coordinates (or anaheim_nodes.geojson)
│   │   └── anaheim_master.gpkg        # Generated by 'scenarios initialize'
│   └── chicago/
│       └── ...
```

#### Required Input File Formats:
- **`net.tntp`**: Standard TNTP network link file containing columns `init_node`, `term_node`, `capacity`, `free_flow_time`, `b` (alpha), `power` (beta).
- **`trips.tntp`**: Standard TNTP OD matrix file with `Origin <id>` blocks.
- **`flows.tntp`**: Tab-separated file with `a_node`, `b_node`, `flow`, `cost`.
- **`nodes.tntp`** or **`<network>_nodes.geojson`**: Node coordinate table (`node_id`, `x`, `y`).

---

### 2. Output Scenario Structure (`data/`)
Scenario generation creates a `scenarios_geojson/` folder under `data/{network}_{a|b}/` structured as follows:

```text
code/
└── data/
    ├── anaheim_a/                     # Dataset A scenarios (Uniform scaling)
    │   └── scenarios_geojson/
    │       ├── scenario_00000/        # Master baseline (unmodified inputs + ground-truth flows)
    │       │   ├── nodes.geojson
    │       │   ├── links.geojson
    │       │   ├── od.geojson
    │       │   └── flows.geojson      # Included only for scenario_00000
    │       ├── scenario_00001/        # Perturbed scenario 1
    │       │   ├── nodes.geojson
    │       │   ├── links.geojson
    │       │   └── od.geojson
    │       └── ...
    └── anaheim_b/                     # Dataset B scenarios (Dirichlet distribution shift)
        └── scenarios_geojson/
            └── ...
```

## Installation & Environment Setup

This package is designed to run in an isolated environment managed by [`pixi`](https://pixi.sh).

### Option 1: Via Pixi (Recommended)
No manual installation required. Use `pixi run` within the project root:

```bash
pixi run -e scenario-generation <command>
```

### Option 2: Standalone Editable Install
In a Python 3.13 virtual environment:

```bash
pip install -e ./libs/scenario-generation
```

---

## Command-Line Usage

### 1. Network Initialization (`initialize`)
Compiles raw TNTP files into a master GeoPackage (`<network>_master.gpkg`):

```bash
# Using Pixi
pixi run -e scenario-generation initialize anaheim

# Direct CLI
scenarios initialize anaheim --path networks
```

#### Arguments & Options:
| Argument / Option | Type | Default | Description |
| :--- | :--- | :--- | :--- |
| `NETWORK` | `TEXT` | *Required* | Name of the network folder under `networks/` (e.g. `anaheim`, `chicago`). |
| `--path` | `TEXT` | `networks` | Base path to the networks directory. |

---

### 2. Scenario Generation (`generate`)
Generates $N$ scenario GeoJSON subdirectories from `<network>_master.gpkg`:

```bash
# Generate Dataset A (Uniform Scaling, 700 scenarios)
pixi run -e scenario-generation generate anaheim -n 700

# Generate Dataset B (Dirichlet Distribution Shift, 700 scenarios, Multiprocessed)
pixi run -e scenario-generation generate chicago -n 700 --dataset-type dataset_b --multiprocess

# Direct CLI usage
scenarios generate anaheim -n 700 --dataset-type dataset_a --multiprocess
```

#### Arguments & Options:
| Argument / Option | Type | Default | Description |
| :--- | :--- | :--- | :--- |
| `NETWORK` | `TEXT` | *Required* | Name of the network (e.g. `anaheim`, `chicago`). |
| `-n`, `--n-scenarios` | `INT` | `5000` | Number of perturbed scenarios to generate. |
| `--dataset-type` | `Choice` | `dataset_a` | Perturbation profile: `dataset_a` (Uniform) or `dataset_b` (Dirichlet). |
| `--multiprocess` | `FLAG` | `False` | Enables parallel scenario generation across CPU cores. |
| `--path` | `TEXT` | `networks` | Base path to search for `<network_name>_master.gpkg`. |
| `--output` | `TEXT` | `data/` | Root directory where scenarios will be saved. |

## Python API Usage

You can also import and use the package programmatically:

```python
from pathlib import Path
from scenario_generation.generate_scenarios import ScenarioGenerator
from scenario_generation.conv_to_gpkg import create_gpkg

# 1. Compile TNTP files to GeoPackage
create_gpkg(network="anaheim", path="networks")

# 2. Instantiate ScenarioGenerator
generator = ScenarioGenerator(network="anaheim", path=Path("networks"))

# 3. Generate 700 scenarios for Dataset B
generator.run(
    n_scenarios=700,
    output_dir=Path("data/anaheim/scenarios_geojson"),
    dataset_type="dataset_b",
    multiprocess=True
)
```
